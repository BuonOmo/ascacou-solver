# Contributing

## Commit message

- Use [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/)
  for commit messages. The `<type>` can be:
  - `feat`: A new feature
  - `fix`: A bug fix
  - `perf`: an improvement in performances, the message
    should include some benchmark result
  - `style`: update in either code style or stdout style.
    No algorithm change.
  - `tests`: adding or updating tests
	- `docs`: updating documentation

## Release process

- [ ] Update the performances benchmark in `README.md`
- [ ] Run tests
